# Улучшение отображения страницы проекта

## Цель
Улучшить информационную архитектуру страницы проекта: убрать лишнее из KPI-ряда, перенести контекст в описание, оптимизировать порядок блоков.

## Изменения

### Бэкенд

**ProjectSummaryDTO** — новое поле `totalSpentHours: number | null`

**getProjectSummary()** — вычисление:
```
totalSpentHours = sum(reports.map(r => r.totalSpentMinutes)) / 60
```

### Фронтенд

**1. Описание страницы**
Заменить статический текст на:
```
{N} сотрудник(а/ов) · Показатели за неделю: {start} — {end}
```
- Плюрализация: 1 сотрудник, 2-4 сотрудника, 5+ сотрудников
- Период из `summary.lastPeriodStart` / `summary.lastPeriodEnd`
- Без периода если данных нет

**2. KPI-карточки**
- Убрать «Сотрудников»
- Добавить «Списано часов»: `totalSpentHours.toFixed(1) ч`, нейтральный цвет, tooltip

**3. Удалить PeriodIndicator** из ProjectPage (компонент остаётся в кодовой базе)

**4. Порядок блоков:**
KPI → График → Concerns → LLM-рекомендации → EmployeeTable
