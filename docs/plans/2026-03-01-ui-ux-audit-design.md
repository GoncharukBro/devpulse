# UI/UX Audit — Design Document

**Дата**: 2026-03-01
**Подход**: Послойный (системные → компоненты → страничные)

---

## Слой 1 — Layout и системные изменения

### 1.1 Одноколоночный layout (Overview, ProjectPage)

Страницы `OverviewPage` и `ProjectPage`: убрать двухколоночный layout `lg:grid-cols-3` (2/3 график + 1/3 алерты). Заменить на:
- График на полную ширину
- ConcernsList под графиком (компактные карточки)

`EmployeePage`: оставить 2/3 + 1/3, добавить `max-h-[400px]` + «Читать полностью» на LLM-сводку.

### 1.2 ConcernsList → компактные карточки

Новый формат алертов:
```
[Avatar] Имя Фамилия — ПроектX     [Score: 42] [→]
         ⚠ Перегрузка, низкая точность
```
- Grid `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`
- Показывать топ-5, кнопка «Показать все (N)»
- Клик → профиль сотрудника

### 1.3 Индикатор периода

На Overview, ProjectPage, EmployeePage — текст под KPI:
```
Данные за последнюю неделю: 24 фев — 2 мар 2026
```

### 1.4 completionRate = 0%

**Диагностика** (бэкенд): разобраться, почему completionRate всегда 0%.
Известно: `MetricsCollector` проверяет `issue.resolved` timestamp из YouTrack API. Если в YouTrack статусы не помечены как "Is Resolved" — поле пустое.

**Фронтенд**: если completionRate = null/0 без данных → показывать «—» вместо «0.0%» красного. Применить на EmployeesListPage, ProjectPage, EmployeePage.

## Слой 2 — Компоненты и графики

### 2.1 Адаптивные графики типов (EmployeePage)

- IssuesByTypeChart (donut): если < 3 типов → горизонтальные бары вместо donut
- SpentByTypeChart (bar): если 1 бар → уменьшить высоту карточки

### 2.2 Спарклайны на карточках проектов и команд (условно)

**Зависит от API**: проверить, отдаёт ли бэк историю score для списков.
Если да → мини-SVG спарклайн (5 точек) на ProjectCard и TeamCard.
Если нет → отложить, не блокировать остальное.

### 2.3 LLM-сводка с ограничением высоты

- `max-h-[400px] overflow-y-auto` на LlmSummaryBlock
- Кнопка «Читать полностью» → раскрытие с анимацией

---

## Слой 3 — Страничные доработки

### 3.1 /employees — таблица

- Закрытие/Тренд = null/0 → «—» серым вместо красного «0.0%»
- Строки кликабельны: `cursor-pointer` + `hover:bg-gray-50` → navigate к профилю

### 3.2 /teams — карточки и страница

Карточки: добавить Точность, кол-во проектов.
Страница: добавить KPI (Точность, Закрытие), секцию «Обратите внимание».

### 3.3 /projects/:id — LLM-рекомендации

Дедупликация на фронте: lowercase + убрать пунктуацию + `includes()`.
Если одна содержит другую — оставить более длинную.
Кнопка «Копировать» к блоку рекомендаций.

### 3.4 /collection

Продублировать «Запустить всё» под карточками.

### 3.5 /achievements — стрики (диагностика)

Проверить логику стриков (70 недель при 5 неделях данных). Отдельная диагностика, не блокирует.

---

## Не входит в scope

- Полный редизайн sidebar
- Новые API-эндпоинты (кроме проверки истории score для спарклайнов)
- Мобильная адаптация (текущая работает)
