DevPulse — Методология
======================

Как работает DevPulse — источники данных, метрики и их расчёт.


1. Как работает система
-----------------------

DevPulse автоматически собирает данные из YouTrack раз в неделю:
задачи, списания времени, статусы. На основе этих данных рассчитываются
ключевые показатели по каждому сотруднику и проекту.

AI-модель анализирует метрики и формирует текстовую сводку
с оценкой, достижениями и рекомендациями.


2. Источники данных
-------------------

Все данные берутся из YouTrack через API:

  • Задачи — тип, статус, исполнитель, оценка, даты создания и закрытия
  • Списания времени (work items) — кто, сколько, когда, тип работы
  • Участники проекта — для определения состава команды

DevPulse только читает данные. Система не создаёт, не изменяет и не удаляет
задачи в YouTrack.


3. Ключевые метрики
-------------------

Загрузка (Utilization)
  Насколько загружен сотрудник относительно 40-часовой рабочей недели.
  Формула: (списанное время / 40 часов) × 100%
  🟢 60–100%   🟡 40–59% или 101–120%   🔴 < 40% или > 120%
  Н/Д: если нет списаний — 0% (валидное значение)

Точность оценок (Estimation Accuracy)
  Насколько точно сотрудник оценивает сроки задач.
  Формула: min(оценка, факт) / max(оценка, факт) × 100%
  🟢 ≥ 75%   🟡 55–74%   🔴 < 55%
  Н/Д: если нет задач с оценкой и фактом

Фокус (Focus)
  Доля времени на продуктовые задачи (фичи, техдолг, документация)
  от общего списанного времени.
  Формула: (время на фичи + техдолг + docs) / общее время × 100%
  🟢 ≥ 65%   🟡 45–64%   🔴 < 45%
  Н/Д: если нет списаний

Закрытие (Completion Rate)
  Процент завершённых задач от общего количества за период.
  Формула: завершённые / всего × 100%
  🟢 ≥ 70%   🟡 50–69%   🔴 < 50%
  Н/Д: если нет задач

Cycle Time
  Среднее время от взятия задачи в работу до её закрытия.
  Формула: среднее(дата закрытия − дата начала работы)
  🟢 ≤ 48 ч   🟡 49–96 ч   🔴 > 96 ч
  Н/Д: если нет закрытых задач с историей статусов

Списано часов
  Суммарное время, списанное сотрудником за неделю (work items из YouTrack).
  Формула: сумма всех списаний за период
  🟢 30–45 ч   🟡 20–29 ч или 46–55 ч   🔴 < 20 ч или > 55 ч
  Н/Д: если нет списаний времени

Score (AI-оценка)
  Общая оценка продуктивности от AI-модели (0–100). Рассчитывается
  на основе всех метрик с учётом контекста.
  Формула: LLM-оценка (0–100)
  🟢 ≥ 70   🟡 50–69   🔴 < 50
  Н/Д: если AI-анализ не выполнен или нет данных


4. Тренды метрик
----------------

Рядом с каждой метрикой отображается индикатор тренда — стрелка и числовая
дельта. Тренд показывает, как изменилась метрика по сравнению с предыдущей
неделей.

  ↑ +5.2%  — значение выросло (выше порога чувствительности)
  ↓ -3.1%  — значение снизилось (ниже порога)
  → 0.8%   — значение стабильно (изменение в пределах порога)
  —        — нет данных за предыдущий период для сравнения

Пороги чувствительности:
  • Score, Загрузка, Точность, Фокус, Закрытие — ±5 пунктов
  • Cycle Time — ±2 часа
  • Списано часов — ±10 часов

Числовая дельта показывается всегда (включая стабильный тренд),
чтобы видеть точное изменение.


5. AI-анализ
------------

После сбора метрик AI-модель получает агрегированные показатели и формирует:

  • Score (0–100) — общая оценка продуктивности
  • Сводка — краткое описание результатов за период
  • Достижения — что сделано хорошо
  • Внимание — проблемы и риски
  • Рекомендации — что можно улучшить

AI анализирует названия задач и определяет их бизнес-значимость.
Оценка учитывает не только цифры, но и характер работы.


6. Система достижений
---------------------

Достижения — это автоматические награды, которые сотрудники получают
за результаты работы. Система анализирует еженедельные метрики из YouTrack
и присваивает награды тем, кто достиг определённых показателей.

Как это работает:
  1. Каждую неделю система собирает метрики по задачам и списаниям времени
  2. По результатам автоматически проверяются условия каждого достижения
  3. Если сотрудник выполнил условие — он получает награду соответствующего уровня
  4. Лучшие результаты фиксируются как рекорды

Уровни наград:
  Common    ★☆☆☆  — базовый результат
  Rare      ★★☆☆  — хороший результат
  Epic      ★★★☆  — отличный результат
  Legendary ★★★★  — выдающийся результат

Чем выше показатель — тем выше уровень награды. Пороги индивидуальны
для каждого типа достижения.

Серии (стрики):
  Если сотрудник получает одно и то же достижение несколько недель подряд —
  формируется серия. Длинные серии показывают стабильность результатов.

Категории:
  ⚡ Продуктивность — объём и скорость выполнения задач
  🎯 Качество — точность оценок, отсутствие багов, скорость закрытия
  ⚖️ Баланс — оптимальная загрузка, фокус на продуктовых задачах
  📈 Рост — положительная динамика показателей


7. Отчёты за произвольный период
---------------------------------

Отчёты позволяют проанализировать работу сотрудника, проекта или команды
за произвольный период — от нескольких дней до нескольких лет. Данные
собираются напрямую из YouTrack за точный указанный диапазон дат.

Как это работает:
  Вы выбираете произвольные даты «от» и «до» — без округления до недель.
  Система выполняет трёхфазный pipeline:

  1. Сбор данных — для каждого сотрудника запрашиваются метрики из YouTrack
     за указанный период: задачи, время, статусы, типы работ

  2. Агрегация — собранные данные группируются, рассчитываются KPI,
     формируется топ-20 задач (гибридная выборка: по времени, просроченные,
     бизнес-критичные)

  3. AI-анализ — двухуровневый LLM-анализ: сначала по каждому сотруднику
     (score, сводка, проблемы, рекомендации, классификация задач), затем
     итоговая сводка по проекту/команде

Уровни отчётов:
  • Сотрудник — метрики одного человека, с разбивкой по проектам
    (если участвует в нескольких)
  • Проект — метрики по всем сотрудникам проекта + индивидуальный
    AI-анализ каждого
  • Команда — метрики по участникам команды из всех их проектов

AI-анализ:
  Для каждого сотрудника AI получает: агрегированные метрики, таблицу
  динамики с типами задач и топ-20 ключевых задач. На основе этого
  формируется оценка (score 0–100), текстовая сводка, список проблем
  и рекомендаций. Для проектов и команд дополнительно формируется итоговая
  сводка на основе индивидуальных анализов.

Статусы:
  ● Сбор данных (синий, пульсирующий) — идёт сбор метрик из YouTrack
  ● Анализ (фиолетовый, пульсирующий) — выполняется AI-анализ собранных данных
  ● Готов (зелёный) — отчёт полностью сформирован
  ● Частично готов (жёлтый) — часть AI-анализов не удалась, метрики доступны
  ● Ошибка (красный) — сбор или анализ не удался

В списке отчётов можно раскрыть любую строку, чтобы увидеть текущее состояние
pipeline и собранные метрики в реальном времени.


8. Совместный доступ
--------------------

Владелец подписки может предоставить доступ другим пользователям по логину
с одной из двух ролей. Это позволяет коллегам работать с метриками без
необходимости создавать собственную подписку на тот же проект.

Роли:
  ┌───────────────────────────────────┬──────────┬──────────┬──────────┐
  │ Действие                         │ Владелец │ Редактор │ Просмотр │
  ├───────────────────────────────────┼──────────┼──────────┼──────────┤
  │ Просмотр метрик и отчётов        │    ✓     │    ✓     │    ✓     │
  │ Запуск и остановка сбора          │    ✓     │    ✓     │    —     │
  │ Редактирование сотрудников        │    ✓     │    ✓     │    —     │
  │ Настройка маппинга полей          │    ✓     │    ✓     │    —     │
  │ Управление доступом               │    ✓     │    —     │    —     │
  │ Включение/исключение из автосбора │    ✓     │    —     │    —     │
  │ Удаление подписки                 │    ✓     │    —     │    —     │
  └───────────────────────────────────┴──────────┴──────────┴──────────┘

Как управлять доступом:
  Откройте меню подписки (три точки) и выберите «Доступ».
  В открывшемся окне введите логин пользователя, выберите роль и нажмите
  «Добавить». Роль можно изменить в любой момент через выпадающий список
  рядом с логином. Для отзыва доступа нажмите кнопку удаления.

Индикация:
  Подписки, к которым вам предоставлен доступ, отмечены бейджем роли
  на карточке проекта: «Просмотр» или «Редактор».


9. Что означает "Нет данных"
----------------------------

Метрика показывает «Н/Д» если:

  • За период нет задач в YouTrack → все метрики = Н/Д
  • Нет списаний времени → загрузка = 0% (это валидно, не Н/Д)
  • Нет задач с оценкой → точность = Н/Д
  • AI-анализ не выполнен → score = Н/Д

Если все метрики = Н/Д, проверьте настройки маппинга полей проекта.
